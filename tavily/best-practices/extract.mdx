## Extracting web content using Tavily

Extracting content from web pages is a core requirement for AI-powered applications. Tavily supports two primary extraction approaches, each suited to a different workflow.

### 1. One-step extraction: directly retrieve `raw_content`

Enable `include_raw_content = true` in a Tavily Search API call to retrieve both search results and extracted content in a single step.

This approach can increase latency because you may extract raw content from sources that are not relevant. For many applications, a two-step process is more efficient: run multiple sub-queries to expand the source pool, then curate the most relevant documents based on content snippets or source scores. Extracting raw content only from the most relevant sources produces higher-quality RAG documents.

### 2. Two-step process: search, then extract

For higher accuracy and greater control, use a two-step process:

<Steps>
  <Step title="Step 1: Search">
    Use the Tavily Search API to retrieve relevant web pages and their URLs.
  </Step>
  <Step title="Step 2: Extract">
    Use the Tavily Extract API to fetch the full content from the selected URLs.
  </Step>
</Steps>

**Example:**

```python
import asyncio
from tavily import AsyncTavilyClient

tavily_client = AsyncTavilyClient(api_key="tvly-YOUR_API_KEY")

async def fetch_and_extract():
   # Define the queries with search_depth and max_results inside each query dictionary
   queries = [
       {"query": "AI applications in healthcare", "search_depth": "advanced", "max_results": 10},
       {"query": "ethical implications of AI in healthcare", "search_depth": "advanced", "max_results": 10},
       {"query": "latest trends in machine learning healthcare applications", "search_depth": "advanced",
        "max_results": 10},
       {"query": "AI and healthcare regulatory challenges", "search_depth": "advanced", "max_results": 10}
   ]

   # Perform the search queries concurrently by passing each query dictionary
   responses = await asyncio.gather(*[tavily_client.search(**q) for q in queries])

   # Filter URLs with a score greater than 0.5. You can also use a re-ranking model, an LLM, or document clustering to identify the most relevant sources
   relevant_urls = []
   for response in responses:
       for result in response.get('results', []):
           if result.get('score', 0) > 0.5:
               relevant_urls.append(result.get('url'))

   # Extract content from the relevant URLs
   extracted_data = await asyncio.gather(*(tavily_client.extract(url) for url in relevant_urls))

   # Print the extracted content
   for data in extracted_data:
       print(data)

# Run the function
asyncio.run(fetch_and_extract())
```

#### **Pros of two-step extraction**

✅ **More control** – Extract content only from selected URLs.

✅ **Higher accuracy** – Filter out irrelevant results before extraction.

✅ **Advanced extraction capabilities** – Use `search_depth = "advanced"`.

#### **Cons of two-step extraction**

❌ Slightly higher cost.

### Using advanced extraction

Use `extract_depth = "advanced"` in the Extract API for more comprehensive content retrieval. This mode is particularly useful when working with:

- **Complex web pages** with dynamic content, embedded media, or structured data.
- **Tables and structured information** that require accurate parsing.
- **Higher success rates**.

<Callout kind="tip">
  If precision and depth are priorities for your application, `extract_depth = "advanced"` is the recommended choice.
</Callout>
