## What is the Tavily Search Engine?

Building an AI agent that uses real-time online information is challenging. Scraping does not scale well, and traditional search APIs often return broad lists of links and snippets that still require filtering and extraction. Tavily was designed to reduce that work.

Tavily is a search engine optimized for LLMs, with fast and persistent search results. Unlike general-purpose search APIs, Tavily is built for AI developers and autonomous agents that need relevant information from online sources in a single API call.

To try the API, use our hosted version in the [API Playground](https://app.tavily.com/playground).

## How does the Search API work?

Traditional search APIs such as Google, Serp, and Bing return results based on a user query, but those results may still require additional scraping, filtering, and context trimming before they are useful in an LLM workflow. Tavily handles that work in a single API call.

The Tavily Search API can aggregate up to 20 sites per request and uses proprietary AI to score, filter, and rank the most relevant sources for your task, query, or goal.
In addition, Tavily supports custom fields such as context and response token limits so you can shape the output for your application.

Tavily can also help your AI agent make better decisions by including a short answer for cross-agent communication.

<Callout kind="tip">
  For RAG workflows, the right context matters. Tavily helps you retrieve focused information that is better suited for downstream model use.
</Callout>
