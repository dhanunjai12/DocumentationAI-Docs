## Overview

The Get Async Chat Completion Response endpoint allows you to check the status and retrieve the results of a previously submitted asynchronous chat completion job. Use this endpoint to poll for completion status and access the final response when processing is complete.

Common use cases:
- Poll for completion status of async chat completion jobs
- Retrieve the final response from completed async requests
- Monitor processing progress of long-running tasks
- Check error messages for failed async jobs

## Endpoint

````http
GET https://api.perplexity.ai/async/chat/completions/{request_id}
````

## Authentication

This endpoint requires Bearer token authentication. Include your API key in the Authorization header:

````http
Authorization: Bearer YOUR_API_KEY
````

## Path Parameters

<ParamField path="request_id" param-type="string" required="true" showLocation="false">
  The ID of the asynchronous chat completion request. This is the unique identifier returned when the async job was created.
</ParamField>

### Error Response (404)

Returns when the specified request ID is not found.

````json
{
  "error": "Async chat completion response not found."
}
````

## Status Values

- **CREATED** - Job has been created but processing has not started
- **IN_PROGRESS** - Job is currently being processed
- **COMPLETED** - Job has been successfully processed and results are available in the response field
- **FAILED** - Job processing has failed, check error_message for details

## Common Patterns

- **Polling for completion** - Call this endpoint at regular intervals until status changes to COMPLETED or FAILED
- **Exponential backoff** - Implement exponential backoff in polling logic to reduce API calls while waiting for long-running jobs
- **Result caching** - Once status is COMPLETED, cache the response to avoid repeated API calls for the same request
- **Error handling** - Check for status FAILED and inspect error_message to handle processing failures appropriately
- **Timestamp tracking** - Use created_at, started_at, and completed_at timestamps to calculate processing duration and monitor performance
