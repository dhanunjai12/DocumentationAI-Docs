## CRITICAL RULES

### Code Examples - ALWAYS USE 4 BACKTICKS
mdx
````jsx
<Component attribute="value">
  Content
</Component>
````

### Component Syntax
**ParamField**: Use showLocation="false" for component props, required/deprecated as strings
**Callout kinds**: info, tip, success, alert, danger
**Card**: Required: title, href. Optional: icon, image, cta, horizontal, target
**Columns**: cols={2} (number, not string) - values: 1, 2, 3, 4, 5
**Expandable**: default-open="true" or "false" (strings)

### Boolean Attributes - ALWAYS AS STRINGS
mdx
✅ <Component required="true" horizontal>
❌ <Component required={true} horizontal={true}>

### Number Attributes - AS NUMBERS
mdx
✅ <Columns cols={2}>
❌ <Columns cols="2">

## CONTENT RULES

### NEVER USE:
Test data: "Test Card 1", "Example Text", "Demo Content"
Placeholder values: "foo", "bar", "test123"
Emojis or casual language
Generic descriptions: "This is for various purposes"

### ALWAYS USE:
Real-world examples: "Getting Started Guide", "API Reference"
Specific descriptions: "Create collapsible FAQ sections"
Professional tone
Meaningful URLs: "/docs/quickstart" not "#" or "/test"

## ERROR PREVENTION

### MDX Syntax Errors
❌ Three backticks in examples → ✅ Four backticks
❌ Missing language identifier → ✅ Always specify: jsx, typescript, etc.
❌ Unclosed tags → ✅ Proper closing: </Component>
❌ Wrong quotes on attributes → ✅ Use double quotes: attribute="value"

### Component Errors
❌ Non-existent attributes → ✅ Only use documented attributes
❌ Wrong attribute types → ✅ Match documented types
❌ Multiple location attributes → ✅ One location per ParamField

### Content Errors
❌ Vague descriptions → ✅ Specific, actionable guidance
❌ Missing examples → ✅ Show code + rendered output
❌ Inconsistent structure → ✅ Follow mandatory structure exactly
